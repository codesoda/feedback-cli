Sample discussion document

This template renders any HTML you place in #doc-content as a commentable document. Every heading, paragraph, list item, blockquote, code block, or stylized box (.quote, .decision, .callout, .context) becomes clickable on load — it gets a stable data-anchor-idx assigned so comments bind to a position that survives page reloads.

Markup conventions understood out of the box

  • Headings h1–h5 — section titles.
  • p, li — standard prose.
  • blockquote — an attributed quote.
  • pre — multi-line code or log blocks.
  • .quote — a highlighted input or question (yellow).
  • .decision — a captured decision (green).
  • .callout — a generic highlighted note (blue).
  • .context — monospace context / data block (grey).

How to use this template

  1. Copy this file somewhere (e.g. /tmp/my-review.html).
  2. Replace #doc-content with the document you want reviewed.
  3. Optionally pre-seed discussion threads via the #prepopulated-threads JSON block below.
  4. Open in a browser. Click any element to comment; drag-select for a range.
  5. Use the Done — send to chat button to emit the transcript and finish the session.
Thread anchors survive reloads because they bind to data-anchor-idx (the 1-based position of the commentable element within #doc-content). If you re-order elements between reviews, previously-saved threads may misalign — start a new server session in that case.

Prepopulated threads

Drop items into the #prepopulated-threads JSON block to seed the review with existing comments (useful when an agent prepares takes for a human to react to). Each entry looks like:

{
  "id": "c1",
  "anchorIdx": 3,          // 1-based position among commentable elements
  "anchorSelector": "h2:nth-of-type(2)",  // optional alternative to anchorIdx
  "index": 1,              // display number (e.g. "Comment 1 of N")
  "subtitle": "§Intro · first paragraph",
  "userComment": "The original reviewer comment we're responding to",
  "take": "<p>My take as HTML. Can include <code>tags</code>.</p>",
  "decision": "One-line decision captured from this thread.",
  "pending": true          // true = yellow marker, 'initial take' notice; false/omit = blue
}
Click a marker in the left margin to open a comment.
Blue = pre-seeded · Yellow = pending · Pink = your thread.

To add your own comment: click any element — or drag-select across multiple elements — then type your note.
